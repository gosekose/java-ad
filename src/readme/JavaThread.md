# 스레드를 직접 사용할 떄의 문제점
- 스레드 생성 시간으로 인한 성능 문제
- 스레드 관리 문제
- Runnable 인터페이스의 불편함

---

## 스레드 생성 시간으로 인한 성능 문제
- 스레드를 사용하려면 스레드를 생성
- 메모리 할당
  - 각 스레드는 자신만의 호출 스택을 가지고 있어야 함
  - 이 호출 스택은스레드가 실행되는 동안 사용하는 메모리 공간
  - 스레드를 생성할 때는 이 호출 스택을 위한 메모리를 할당해야 함
- 운영체제 자원 사용
  - 스레드를 생성하는 작업은 운영체제 커널 수준에서 이루어짐
  - 시스템 콜을 통해 처리
  - CPU와 메모리 리소스를 소모하는 작업
- 운영체제 스케줄러 설정
  - 새로운 스레드가 생성되면 운영체제의 스케줄러는 이 스레드를 관리하고 실행 순서를 조정해야 함
  - 운영체제의 스케줄링 알고리즘에 따라 오버헤드 발생
  - 스레드는 하나의 보통 1MB 이상의 메모리를 사용

---

## Runnable 인터페이스의 불편함
- 반환값이 없음
  - 스레드의 반환 값을 받을 수 없음
- 체크 예외를 던질 수 없음, 체크 예외는 메서드 내부에서 처리해야 함

---

## 스레드 풀
- 스레드를 관리하는 스레드 풀에 스레드를 미리 필요한 만큼 만듬
- 스레드는 스레드 풀에서 대기하며 쉼
- 스레드 풀에서 스레드가 관리되기 때문에 필요한 만큼만 스레드를 만들 수 있고, 관리할 수 있음
